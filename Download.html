Download CPL

The CPL tools come in the form of a self-extracting archive, which you can download at the bottom of this page. Installation in user space requires no privileges. To install, run the downloaded file in a terminal as "sh install-cpl.sh" and follow the instructions on screen.

System requirements

The CPL compiler and interpreter run on Linux/unix systems. They have been developed on Debian GNU/Linux, and this is the primary system suggested for their usage. Nevertheless, the installation is not tied to a specific flavour of Linux or unix nor to a specific processor architecture. Although incompatibilities can arise from variations in the system C library (your mileage may vary), installation has been successful in some versions of the MacOS Xcode environment, as well as in Raspberry Pi and Beaglebone Black ARM-based microboards. And the Windows Subsystem for Linux (WSL), which is an integral part of Windows 10, essentially is Debian GNU/Linux.

Only basic system tools are required: gzip, tar, cc, make, info, plus gnuplot if you want graphics. For maximal compatibility the system compiler should be gcc (as it is by default in Debian), but other compilers (e.g. icc) have also been used. However, CPL nested functions are implemented as C nested functions, a nonstandard gcc extension, and will become unavailable if a compiler other than gcc is used (or if the system's gcc disables them).

Code maturity

CPL is under active development; its portions sit at different levels of maturity. On a crude scale:

makecpl has been under development for decades. It now only undergoes unfrequent changes.
icpl is best classified as experimental. Don't be surprised if it does not always behave as intended, but it can be very useful when it does.
edit is even younger. It may on rare occasions crash and lose your edits, so be warned to save often (as everybody should do anyway). Nevertheless it is the editor of choice for the maintenance of CPL itself, writing this website included.

On a finer scale, the optional CPL libraries provided in the package also come with different levels of maturity. See comments in each.

Mature or not, bugs will happen. If you believe you found a bug, or an inconsistency between the compiler's behaviour and its documentation, you will do an appreciable service to the CPL community by filing a bug report with With your description please enclose a code snippet, as short as possible but complete enough that it actually compiles and triggers the non-conforming behaviour.

Version updates are denoted by their date. File names in the version archive include dates.
