Compiler and Programming Language FAQ

Does your programming language use brackets like in Mathematics?

CPL uses brackets just as you were taught in school. All three kinds of round, square, and curly brackets are interchangeably available for grouping arithmetic (or other) expressions. While to alternate parentheses, brackets and braces is optional, to do so eases the human eye's reading and proofreading nested formulas and enhances automatic error catching.

The provided text editor can automatically match and alternate bracket styles for you.

Is this an object-oriented language?

In its own way. CPL avoids the concept of function definitions contained inside classes in favour of polimorphic functions that are specified by the (often STRUCTURE) type of one or more than one of their arguments. It also caters for subtyping and inheritance through anonymous STRUCTURE fields, and for dynamic object typing through DYNAMIC pointers.

Do you offer symbolic manipulation?

The symbolic.cpl library adds symbolic differentiation capabilities to CPL. Symbolic differentiation is seamlessly integrated in the compiler, and is performed at compile time: any desired symbolic derivatives are worked out during compilation, and inlined in the generated code.

What about linear algebra and graphics?

The CPL compiler's most distinctive feature is its incremental extensibility. The included libraries thus provide matrix operations, gnuplot graphics, and symbolic manipulation, just to name three examples, all blended into the language syntax.

Additionally, any function provided by an external C or FORTRAN library can be seamlessly called in a CPL program with no performance penalty.

Is variable array size supported?

CPL arrays allow variable (run-time specified) lower and upper bounds wherever they allow constant ones; the compiler statically or dynamically allocates memory space as needed and frees it on exit from the enclosing code block.

Bounds can be queried through the LO and HI builtin functions, and are run-time checked by the rtchecks module.

A rich set of subarray selections, commuting with structure fields, is efficiently supported.

Are function prototypes enforced?

Function prototypes exist in most languages and allow many programming errors to be caught at compile time. They are also essential to disambiguating overloaded functions. Inconveniently C and FORTRAN and many other compiled languages require dedicated header or interface files that duplicate the definitions contained in the program proper.

CPL checks prototype coherence at compile time by accessing the USEd files as they were headers, and issues compilation errors if needed. The prototype-enforcing process is totally transparent to the user who no longer needs to write separate header files.

Is your programming language compiled?

CPL was thought out as a compiled language from the very beginning; when used for computationally intensive tasks, it inherits the speed of compiled C. The basic cpl make command, used as 
          cpl make <program>[.cpl] [<option>]
is all that is needed to compile and link the source file <program>.cpl into the executable file <program> , which you can then run by its name just like any other executable. As an alternative, the straight cpl run command compiles and then runs <program> in one shot.

Do you provide an interactive environment?

The interactive CPL interpreter accepts expressions, statements or commands at a console prompt. Any CPL or C or FORTRAN library can be loaded into the interpreter and have its functions executed from the command prompt, either alone or embedded in a CPL command or expression. Unix-shell commands are transparently accepted as well.

Do you provide a code editor?

CPL includes a text editor for program development, equipped with word and syntax completion at the touch of the Tab key. Interactive cpl and the editor are actually a single integrated environment: its cpl and editor views can be swapped at the touch of a key. Both tools display in a terminal with its minimalist graphical appearance, but in exchange allow for swift use in a remote connection, and even in a smartphone's ssh client if in a pinch one feels the need for it.
